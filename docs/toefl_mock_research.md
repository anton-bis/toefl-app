## 1. 为什么要做这个软件

### 1.1 最直接的原因

**网上找不到好用的、免费的、离线可用的托福练习工具。**

市面上的现有工具存在一个共同问题：**功能杂乱、体验割裂、不够贴近真实考试**。对于只想安静做题的学生来说，这些工具本身就是一种干扰。

### 1.2 目标用户是谁

| 用户类型 | 典型特征 | 核心需求 |
|----------|---------|---------|
| **自学型考生** | 高中/大学生，独立备考，预算有限 | 免费、贴近真考、无广告 |
| **薄弱题型突破者** | 已有基础，针对性提升某一模块 | 精准反馈、错题标记、薄弱分析 |
| **培训班教师** | 给学生布置练习作业 | 题库管理、学生数据追踪 |

### 1.3 现有工具的真实问题

| 工具 | 能用 | 不能忍 |
|------|------|--------|
| **ETS官方TPO** | 真题、真实 | 收费$50-80/套、需联网、界面老旧 |
| **KMF考满分** | 免费、题库大 | 广告多、功能冗余、手机端体验差 |
| **新东方在线** | 有教师辅导 | 臃重、操作复杂、不够聚焦 |
| **纸质材料** | 无干扰 | 无即时反馈、无法追踪进度 |

**我们的核心差异化优势**：
1. 免费 + 离线可用
2. 苹果风简洁界面，零广告零干扰
3. 填空题逐字母精准反馈（这是其他工具都没有的）
4. 从阅读模块起步，逐步扩展到听说读写全模块

## 2. 产品发展阶段定义

### 阶段1：静态HTML验证版（当前阶段）

| 维度 | 说明 |
|------|------|
| **技术** | Node.js 脚本解析 Markdown → 生成静态 HTML |
| **范围** | 仅阅读模块（2个Module，4种题型） |
| **存储** | 浏览器 localStorage，无账号系统 |
| **目标** | 验证核心交互逻辑（填空题输入、计时器、Review面板、自动保存）是否可行 |
| **风险** | 无法扩展题型、无法动态更新题库、无云端同步 |

**这个阶段的价值在于**：用最简单的技术栈快速验证"这个产品到底好不好用"，避免在复杂架构上浪费时间验证交互体验。

### 阶段2：正式软件（最终目标）

| 维度 | 说明 |
|------|------|
| **技术** | 前端 Vue3/React + 后端 Node.js/Express + 数据库 MongoDB |
| **范围** | 全模块（阅读 + 听力 + 口语 + 写作） |
| **存储** | localStorage 离线缓存 + 云端数据库同步 |
| **用户系统** | 账号登录，跨设备数据同步 |
| **题库管理** | 内置题库管理界面，支持用户自主添加/编辑/导入 |
| **进阶功能** | 错题本、薄弱分析、学习计划、个性化推荐 |

## 4. 用户痛点清单（从真实体验出发）

### 4.1 使用现有工具时的具体糟心事

| 痛点编号 | 痛点描述 | 我们的解决方案 |
|---------|---------|--------------|
| P-01 | "广告弹窗打断做题思路" | 零广告、零弹窗、全屏沉浸 |
| P-02 | "界面和真考试完全不像" | 像素级还原TOEFL iBT界面 |
| P-03 | "只知道对错，不知道错在哪" | 填空题逐格标红标绿，选择题显示正确选项 |
| P-04 | "关浏览器进度全没了" | 自动保存，刷新秒恢复 |
| P-05 | "补全单词只能整个词重做" | 逐字母检查，只标错的那一格 |
| P-06 | "换了设备数据丢失"（正式版解决） | localStorage + 云端同步 |
| P-07 | "题库旧，想加新题加不了"（正式版解决） | 内置题库管理界面 |

### 4.2 情感需求

| 需求 | 场景 | 设计启示 |
|------|------|---------|
| **信心** | 考前模拟，需要确信自己准备好了 | 真实计时、真实界面、真实进度条 |
| **掌控感** | 知道自己哪里薄弱、哪里强 | 结果页KPI、错题统计、标记功能 |
| **专注** | 做题时不想被打断 | 极简UI、无通知、隐藏计时器选项 |
| **进步感** | 想看到自己的提升轨迹 | 历史记录、正确率趋势（正式版） |

## 5. 技术可行性评估

### 5.1 验证版技术评估

| 技术方案 | 评估 | 风险 |
|---------|------|------|
| Markdown解析 → 静态HTML | ✅ 可行，已部分实现 | 模板占位符可能错位 |
| localStorage持久化 | ✅ 可行，已实现基础功能 | 隐私模式下不可用 |
| 填空题逐字母交互 | ✅ 已实现 | 无 |
| 跨页计时器 | ✅ 已实现 | 浏览器长时间未关闭可能溢出 |

### 5.2 正式版技术选型建议

| 层级 | 推荐方案 | 备选方案 | 理由 |
|------|---------|---------|------|
| **前端** | Vue 3 + Vite | React + Next.js | Vue学习曲线平缓，适合轻量项目 |
| **后端** | Node.js + Express | Java Spring Boot | 前后端同语言，减少沟通成本 |
| **数据库** | MongoDB | MySQL | JSON天然适配，适合灵活题库 |
| **离线缓存** | Service Worker + IndexedDB | localStorage | 更大容量，支持结构化数据 |
| **部署** | Vercel/Netlify + Railway | 自建服务器 | 低成本启动，自动CI/CD |

## 6. 风险与缓解

| 风险 | 影响 | 概率 | 缓解方案 |
|------|------|------|---------|
| **题库质量不足** | 产品再好，题不对也没用 | 高 | 优先使用官方退役真题验证 |
| **验证版 → 正式版迁移成本** | 两套架构功能需重写 | 中 | 验证阶段只验证交互，不写业务逻辑 |
| **数据同步一致性** | 多设备数据冲突 | 中 | 以服务端为准，客户端乐观更新 |
| **用户增长后的服务器压力** | 正式版上线后并发问题 | 低（初期） | 使用无服务器架构，按需扩展 |

## 7. 产品定位声明

> **TOEFL练习软件**是一款从阅读模块起步、逐步覆盖全模块的轻量级托福练习平台。验证版用于快速确认核心交互体验，正式版将提供全模块练习、账号同步、题库管理、错题追踪功能，在保留"免费、离线、纯净"核心优势的同时，解决扩展性和个性化需求。
