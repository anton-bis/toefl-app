## 1. 产品概述与目标用户

### 1.1 功能定位

英文打字练习是托福模考系统的一个 Skills 子功能，旨在帮助考生系统性地**养成英文打字格式规范**（空格、大小写、标点），并逐步提升打字速度，为写作模块中 17 分钟 / ~400 词的实际输出场景做准备。

### 1.2 解决的问题

- 中文母语者在英文打字时，空格、大小写、标点后空格等规范尚未形成肌肉记忆
- 限时写作中，打字执行占据过多认知带宽，挤压内容构思的空间
- 现有托福备考工具（ETS 官方 / KMF / 新东方）均不提供专项打字训练

### 1.3 目标用户

| 类型 | 特征 | 与打字功能的关系 |
|------|------|----------------|
| 自学型考生 | 预算有限，独立备考 | 写作时常因打字格式失分而不自知 |
| 写作薄弱者 | 有内容但写不完 | 打字速度是隐性瓶颈 |
| 培训班教师 | 给学生布置作业 | 可将打字练习作为写作前热身环节 |

## 2. 用户流程

```
侧边栏 Skills → 选择「英文打字练习」
  └→ 难度选择页（文章列表，三区折叠卡片，默认展开）
       └→ 点击某篇文章卡片
            └→ 打字练习页
                 ├→ 展示灰色原文 + 光标高亮当前字符
                 ├→ 计时器自动开始（动态时长按词数计算）
                 ├→ 用户直接覆盖输入（正确→黑，错误→红）
                 ├→ [暂停] 冻结计时器，内容保留可见
                 ├→ [重试] 重置当前文章
                 └→ 计时结束 或 完成全部输入
                      └→ 结果页（分层指标）
                           ├→ [重试] 同一篇文章再来一次
                           └→ [返回列表] 回到难度选择页
```

│  └──────┘ └──────┘ └──────┘    │
│  ┌──────┐ ┌──────┐ ...         │
│  │ ...  │ │ ...  │             │
│  └──────┘ └──────┘             │

- 默认三个区全部展开

### 4.3 打字练习页（核心交互界面）

设计参考 Monkeytype / keybr.com 的模式：

```
┌──────────────────────────────────────┐
│                              07:32   │  ← 右上角计时器胶囊（小号）
│                                      │
│                                      │
│  The sun rises in the east and se_   │  ← 灰色原文
│   ↑↑↑    ↑↑↑            ↑↑↑    ↑    │    正确已输 = 黑色
│  已正确  当前高亮            尚未输入   │    错误输入 = 红色 + 下划线
│                                      │    当前待输 = 下划线高亮
│                                      │
│                          [暂停] [重试] │  ← 底部操作按钮
└──────────────────────────────────────┘
```

**交互细则：**

| 场景 | 表现 |
|------|------|
| 初始状态 | 全文灰色，第一个字符有下划线光标高亮 |
| 输入正确字符 | 字符变为黑色，光标前进到下一字符 |
| 输入错误字符 | 字符变为红色（可加删除线或下划线标识），光标不前进 |
| 再次输入错误位置 | 若当前字符已是红色，继续输入无效，光标保持原位 |
| Backspace | 回退一个字符，将其恢复为灰色，光标回退一位 |
| 输入空格/标点 | 与普通字符同等检测，不一致即标红 |
| 计时结束 | 当前字符锁定，弹出结果 |
| 全部完成 | 所有字符打完，自动弹出结果 |

**无独立的输入框/textarea**，整个文本区域直接响应键盘输入。

## 5. 交互规范

### 5.1 打字输入

| 规则 | 说明 |
|------|------|
| 输入源 | 全局 `keydown` + `keypress` 事件捕获，映射到字符位置 |
| 光标表示 | 当前待输入字符下方显示下划线（或浅色底色高亮） |
| 错误处理 | 错误字符标红，用户可继续往后输入（不阻塞） |
| 修正方式 | Backspace 回退到错误位置重新输入 |
| 跨行处理 | 自然换行，光标在行末自动换到下一行首 |

### 5.2 计时规则

| 难度 | 速率基准 | 计时公式 |
|------|---------|---------|
| Beginner | 不限速 | 默认 5 分钟硬上限 |
| Intermediate | ~35 WPM | `ceil(wordCount / 35 * 60)` 秒 |
| Advanced | ~45 WPM | `ceil(wordCount / 45 * 60)` 秒 |

计时器显示格式 `MM:SS`，最后 10 秒文字变红闪烁。

### 5.3 暂停机制

- 点击暂停 → 计时器停止，文本区域变为只读（淡色遮罩或降低透明度提示）
- 点击继续 → 计时器恢复，文本区域重新可编辑
- 暂停期间内容完全可见

### 5.4 折叠机制

- 点击难度标题栏 → 切换该区的展开/收起状态
- 展开状态：显示该区所有卡片
- 收起状态：仅显示难度标题栏 + 文章总数
- 默认全部展开
- 动画过渡 200ms

## 6. 数据模型

### 6.1 语料 JSON 结构

文件：`assets/questions/typing/corpus.json`

```json
[
  {
    "id": "typing-b-001",
    "title": "The Solar System",
    "difficulty": "beginner",
    "wordCount": 68,
    "content": "The solar system is a vast place. It contains the sun and eight planets. Mercury is the closest planet to the sun. Earth is the third planet from the sun. It is the only planet known to support life."
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识，格式 `typing-{难度缩写}-{编号}` |
| title | string | 文章标题，用于卡片展示 |
| difficulty | string | `"beginner"` / `"intermediate"` / `"advanced"` |
| wordCount | number | 词数 |
| content | string | 完整原文（含标点和大小写） |

## 7. 技术栈

| 层面 | 方案 | 理由 |
|------|------|------|
| 模块架构 | `src/modules/skills/typing/index.js` | 与四大模块一致，统一路由注册 |
| 页面渲染 | 原生 JS + DOM 操作 | 项目无框架，沿用现有模式 |
| 路由 | `src/core/router.js` 注册 `/skills/typing` | 复用现有 HashRouter |
| 语料加载 | `fetch('assets/questions/typing/corpus.json')` | 一次性加载，60 篇约 30-50KB |
| 数据持久化 | localStorage | 第一阶段统一方案 |
| 时间计算 | `Date.now()` 差值，不做 `setInterval` 密集更新 | 避免漂移 |
| 图标 | Font Awesome `fa-keyboard`（侧边栏） | 复用现有 CDN |
| CSS 设计令牌 | 沿用 `index.html` 的 `:root` 变量 | 保证视觉统一 |

## 8. 非功能性需求

| 需求 | 指标 |
|------|------|
| 输入响应延迟 | < 16ms（单帧内） |
| corpus.json 体积 | < 50KB（60 篇全文） |
| 页面加载 | < 200ms（含 JSON fetch） |
| 外部依赖 | 无新增（Font Awesome 复用已有） |
| localStorage 占用 | < 100KB（历史记录按 500 次练习估算） |
| 错误检测精度 | 100%（无漏判空格/大小写/标点） |

## 9. 范围外事项

| 事项 | 原因 |
|------|------|
| 多人竞速/对战 | 偏离托福备考工具定位 |
| 语音输入 | 不涉及口语练习 |
| 自定义上传文章 | 第一阶段控制语料质量 |
| AI 自动生成文章 | 语料由人工收集和审核 |
| 云端同步 | 第二阶段整体规划中包含 |
| 音效反馈 | P2 可选优化，非核心体验 |
