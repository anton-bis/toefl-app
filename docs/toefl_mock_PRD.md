## 1. 产品愿景

**TOEFL练习软件**是一款从阅读模块起步、逐步覆盖全模块的轻量级托福备考平台。

验证版（当前阶段）用于快速确认核心交互体验是否好用；正式版将在此基础上升级为动态应用架构，支持全模块练习、账号系统、云端同步和题库管理。

> **一句话定义**：让每个托福考生都能免费使用一款"看起来像真考试、用起来零干扰、做完后精准反馈"的练习工具。

## 2. 目标用户

### 2.1 核心用户画像

**画像1：自学冲刺型 — "小明"**
- 高三学生，3个月后考试，目标阅读25+
- 预算有限，不愿花$80买TPO
- 需要：安静做题环境、真实计时、做完能看到哪里错了
- 当前工具不满：界面不像真考试、广告太多、填空题没有逐字母反馈

**画像2：薄弱突破型 — "小芳"**
- 大二学生，上次阅读22分，想提到27
- 需要知道自己哪类题型弱，针对性练习
- 需要：错题标记、做完后逐题分析、能回到错题重做
- 当前工具不满：只知道对错总数、不告诉我具体哪个字母写错了

**画像3：布置作业的培训老师（正式版）**
- 需要给学生布置练习、查看完成情况
- 需要：题库管理后台、学生进度追踪

## 4. 用户流程

### 4.1 验证版考试流程

```
开始页
  └→ [开始] → Module 1 介绍页 (计时器 11:30)
                └→ [开始] → 任务1 填空题 (Q1-10)
                           └→ [下一题] × 10
                           └→ 任务2 填空题 (Q11-20)
                           └→ 任务3 Email (Q21-22)
                           └→ 任务4 Email (Q23-25)
                           └→ 任务5 Text Chain (Q26-28)
                           └→ 任务6 Academic (Q29-33)
             → Module 2 介绍页 (计时器 9:00)
                └→ [开始] → 任务1 填空题 (Q1-10)
                           └→ 任务2 Academic (Q11-15)
             → 结果页
                └→ [重新开始] → 回到开始页
```

### 4.2 Review流程（任何题目页面）

```
[复查] → 侧面板滑出
  └→ 题号网格（颜色区分状态）
  └→ 点击题号 → 跳转到该题
  └→ [全部标记] / [清除标记] / [跳转到标记题]
  └→ [关闭] → 面板收起
```

### 4.3 核对答案流程

```
[核对答案] → 展开答案核对区
  └→ 统计：总题/正确/错误/正确率
  └→ 逐题：对/错/空 + 正确答案提示
  └→ [只重做错题] → 清空错误答案
  └→ [关闭] → 收起核对区
```

### 4.4 正式版扩展流程

```
登录页 → 仪表盘（选择模块/查看进度）
  └→ 阅读 → 同验证版流程
  └→ 听力 → 音频播放 → 做题 → 结果
  └→ 口语 → 录音 → 倒计时 → 提交
  └→ 写作 → 文本输入 → 计时 → 提交
  └→ 错题本 → 按题型筛选 → 重做
  └→ 薄弱分析 → 图表 → 推荐练习
```

## 5. UI/UX规格

### 5.1 设计令牌

```css
:root {
  --apple-bg: #f5f5f7;          /* 页面背景 */
  --apple-surface: #ffffff;      /* 卡片背景 */
  --apple-text: #1d1d1f;        /* 主文本 */
  --apple-text-muted: #86868b;  /* 次要文本 */
  --apple-primary: #007aff;      /* 强调/主按钮 */
  --apple-success: #34c759;      /* 正确 */
  --apple-warning: #ff9500;      /* 标记 */
  --apple-error: #ff3b30;        /* 错误 */
  --apple-radius: 12px;          /* 圆角 */
  --apple-shadow: 0 4px 20px rgba(0,0,0,0.08);  /* 阴影 */
  --apple-font: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
}
```

### 5.2 布局原则

1. **无页面级滚动条** — 仅内容区域内部滚动
2. **100%视口利用** — 充分利用屏幕空间
3. **阅读题双栏** — 文章左 | 题目右（50/50）
4. **响应式断点** — 768px（平板）、480px（手机）

### 5.3 交互规范

| 交互 | 规范 |
|------|------|
| 按钮 hover | 微上浮 + 阴影加深 |
| 卡片 | 1px边框 + 柔和阴影 |
| 输入框 focus | 底部边框变色 + 加粗 |
| 过渡动画 | 150-250ms，避免跳变 |
| 弹窗确认 | 用户关键操作前需确认（开始考试、重新开始） |

## 6. 数据状态设计

### 6.1 localStorage Schema（验证版）

| Key | 类型 | 示例 | 说明 |

```javascript
// 用户

  _id: ObjectId,
```

## 7. 技术约束

| 约束 | 验证版 | 正式版 |
|------|--------|--------|
| 架构 | 静态HTML，无后端 | 前后端分离 |
